## 欠損値（NaN）とは

CSVの空欄は、pandasに読み込むと **NaN**（Not a Number）という特別な「値がない」マーカーになる。
これは `None` とは別の、pandas/numpy 専用の欠損マーカー。

```python
df.isna()              # 各セルが欠損かの True/False 表
df.isna().sum()         # 列ごとの欠損件数
df[df.isna().any(axis=1)]   # 欠損を含む行だけ抽出
```

## 学べること

| トピック | ひとことで言うと |
|---------|----------------|
| `isna()` / `notna()` | 欠損かどうかの判定 |
| `.sum()` との組み合わせ | 列ごとの欠損件数を数える |
| `.any(axis=1)` | 行方向に「どれか1つでも該当するか」 |
| `dropna(subset=[...])` | 指定した列に欠損がある行を削除 |
| `fillna(値)` | 欠損を値で埋める |
| `duplicated()` | 重複行の検出（デフォルトは全列一致） |
| `drop_duplicates()` | 重複を除去（最初の1件を残す） |
| `duplicated(subset=[...])` | 特定の列だけで重複を判定 |
| `keep=False` | 重複ペアの両方を表示する |

## 欠損値対処の判断フレーム

対処は「削除する」か「埋める」の2択だが、**列の意味で判断**する。このデータでの例:

| 列 | 欠損の意味 | 対処方針 |
|----|-----------|---------|
| `customer_id` | 誰の注文か不明 | 行ごと削除（集計に使えないため） |
| `quantity` | 数量の記入漏れ | 1で補完（最小注文数と仮定） |
| `unit_price` | 単価の記入漏れ | 平均単価で補完（概算として扱う） |

「なんとなく0で埋める」は危険（平均計算が狂う代表例）。方針とその理由をコードコメントに残す習慣も実務では大事。
